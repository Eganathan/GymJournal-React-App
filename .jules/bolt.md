## 2024-05-18 - Nested array lookups inside list mapping loops
**Learning:** React rendering loops mapping over list state frequently perform nested `.find()` iterations over smaller reference arrays like `categories` or `equipment`, resulting in O(N*M) complexity. This codebase pattern appears when cross-referencing display data directly within the component's render function, common in search list screens.
**Action:** When identifying iterations within the `render` path, aggressively identify small reference arrays that can be memoized into `Map` structures indexed by an ID field. This ensures subsequent ID retrievals within loops are O(1) complexity, optimizing the rendering process across re-renders. Use `useMemo` so that the `Map` is only generated when the reference arrays update.
